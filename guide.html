The ProblemAI costs don't stay flat. They grow.

Every new cron job, every new agent, every new feature adds cost. Without tracking, you don't find out until the bill shocks you.

What was happening in my system

I went from 8 agents to 12 and 24 crons to 37 in a few weeks. My costs were scaling with every feature I shipped, but I had no visibility into any of it. When I finally looked:

  • Apify web scraping. Found ways to do most of it for free with the Instagram Graph API and YouTube Data API. I only use Apify now for the things I can't replicate.
  • 30+ API calls going through the wrong provider. I'd set up OpenRouter thinking I'd test different models. But once I committed to Anthropic as my primary, the system kept routing calls through OpenRouter anyway, without even checking if Anthropic was available. I was paying OpenRouter markup on calls that should have gone direct.
  • Sonnet routing every Telegram message. My agent router was using Claude Sonnet to decide which specialist should handle each message. A simple classification task. Haiku does the same thing for 73% less.
  • Crons running every 30 minutes when nothing changed. My collab reply checker was burning tokens checking an empty inbox.
The real problem wasn't what I was wasting today

It was that costs were growing with every feature and I had no way to see it. Without the tracker, I would have hit $200+ within a couple months and had no idea why. The tracker I'm giving you is exactly what I built to catch this before it scaled.

What's IncludedThe AI Cost Tracker

Five TypeScript files you drop into any project. No database, no third-party service, no setup complexity.

File What it does
tracker.ts Logs every LLM call to a local JSON file with model, tokens, cost, and a label you choose
wrapper.ts Drop-in Anthropic client that tracks automatically. Replace one import and you're done
pricing.ts Cost matrix for every major model (Anthropic, OpenAI, Google, Deepseek). Update when prices change
summary.ts Daily summary with anomaly detection. Flags anything that's 2x+ its normal spend
cli-audit.ts Scans your usage and tells you which tasks could use cheaper models
Works with any provider

The wrapper is Anthropic-native, but the tracker works with any LLM. OpenAI, Google, Deepseek, local models. Just call trackUsage() after any API call.

How to set it upEverything happens in Claude Code

Open your project in Claude Code and paste each prompt below. Claude handles the rest. You don't need to know where files go or how imports work.

Prompt 1: Install the tracker

I want to add AI cost tracking to this project. Here are the ai-cost-tracker source files: [drag and drop the src/ folder into the chat, or paste the GitHub link] Add these files to my project, install any dependencies they need, and initialize the tracker in my main entry point with a monthly budget cap of $[your budget]. Make sure the usage log file is in my .gitignore so it doesn't get committed.

Prompt 2: Start tracking your AI calls

Find every place in my codebase where I'm making AI API calls (Anthropic, OpenAI, Google, or any LLM provider) and add cost tracking to each one using the ai-cost-tracker. Give each call a descriptive label based on what it does, like "classify-email" or "draft-reply" or "generate-summary". I want to be able to see exactly which features cost what.

Labels are how you find the waste

The more specific your labels, the more useful your reports. "classify-email" tells you something. "default" tells you nothing.

Once you've been running for a bit, use these prompts to get your reports.

Prompt 3: Daily cost summary

Run the ai-cost-tracker daily summary and show me the results.

This shows you:

  • Today's spend vs your 7-day average
  • Warning if you're above normal
  • Anomaly alerts for any feature that spiked
  • Spend breakdown by feature and by model

Prompt 4: Find where you're overspending

Run the ai-cost-tracker model audit and tell me which of my AI calls could use cheaper models without losing quality.

This scans your usage and finds:

  • Features using expensive models for simple tasks
  • High-frequency callers that could be batched or reduced
  • Estimated dollar savings per month

Prompt 5: Set up budget alerts

Take it furtherAutomate your cost reports

The prompts above are manual. You run them when you remember. But the real power is when your system tells you automatically. Here's what that looks like.

Automated weekly breakdown

Imagine getting a message every Monday morning that says: "Last week you spent $12.40 across 847 calls. Your content-strategy cron is 2x its normal cost. Your message router could save $3/month on Haiku."

That's a cron job that runs the summary and sends it to you on a schedule. No remembering, no manual checks.

Prompt 6: Set up a recurring cost report

Set up a cron job that runs the ai-cost-tracker summary automatically every [week/day] and sends me the report on [Telegram/Slack/email]. Include anomaly detection so I only get alerted if something spikes. Here's my webhook/token: [paste it]

Monthly cost agent

You can also set up an AI agent that reviews your full month of usage, compares it to last month, and gives you a plain-English breakdown of what changed, what's trending up, and what to do about it. This is what I run on my own system.

Prompt 7: Build a monthly cost review agent

Build me a monthly cost review agent that reads my ai-cost-tracker usage log, compares this month to last month, and sends me a plain-English summary of: total spend change, which features cost more or less than before, which models I'm using most, and the top 3 things I should change to save money. Run it on the 1st of every month and send the report to [Telegram/Slack/email].

ReferenceWhere to start with model selection

The single biggest cost saver. Most people use expensive models for everything. Here's where I'd start, but always test with your actual tasks. A model that works for my classification might not work for yours.

Task type Start with Potential savings
Classifying, routing, tagging Haiku up to 73% vs Sonnet
Scoring, ranking, filtering Haiku up to 73% vs Sonnet
Parsing, extracting structured data Haiku up to 73% vs Sonnet
Simple summarization Haiku / 4o-mini up to 73-94%
Drafting content, emails Sonnet up to 80% vs Opus
Creative writing, strategy Sonnet up to 80% vs Opus
Complex reasoning, code gen Sonnet or Opus test both
How to actually decide

Run the same 10 prompts through both models. Compare the outputs. Ask yourself:

  • Is the cheap model's output good enough for this task?
  • Does the expensive model fail less often (fewer retries = lower real cost)?
  • How many times per day does this run? A $0.001 difference at 100 calls/day is $3/month.

The cheapest model isn't always the cheapest option. A model that needs 3 retries costs more than one that gets it right the first time. Let the data decide, not the price tag.
